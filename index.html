01 Overview

What is RoRanks?

RoRanks is a ranking service built for Roblox groups. Instead of managing your own Roblox cookie, dealing with rate limits, and writing retry logic from scratch, you make a single HTTP request and RoRanks takes care of everything. That includes a global queue, automatic retries, and a Discord webhook notification for every ranking attempt.

All requests go to https://api.roranks.com and all bodies are JSON. Always send the Content-Type header set to application/json.

02 Authentication

API Keys

Every public endpoint needs a valid API key. You pass it in the request body as the key field. Your key is tied to a specific Roblox group, a usage quota, and an expiry date.

⚠ Keep your key secret. Anyone who has it can rank users in your group. Never put it in a LocalScript or anything client-side. Always call from a server Script.

How to get a key

Keys are handed out by the RoRanks team. Join discord.gg/tmGheDXbeP and request one. Once it is created you will get a UUID key string, an expiry date, and separate monthly quotas for App and Pass operations.

Returns whether a Roblox user is blacklisted from being ranked in your group. Call this before any ranking request so blacklisted players get stopped early.

Request body

FieldTypeRequiredDescription
keystringREQUIREDYour API key
useridstringREQUIREDThe Roblox User ID you want to check
 
{ "blacklisted": false }
// or if they are blacklisted

💡 If you get back "blacklisted": true skip the ranking call and let the player know they cannot be ranked.
Luau showing a full blacklist check then rank flow
local function rankUser(userId, rank)
  -- Check if the player is blacklisted first
  local bl = HttpService:RequestAsync({
    Url    = "https://api.roranks.com/blacklists",
    Method = "POST",
    Headers = { ["Content-Type"] = "application/json" },
    Body   = HttpService:JSONEncode({ key = API_KEY, userid = tostring(userId) })
  })
  if HttpService:JSONDecode(bl.Body).blacklisted then
    return false, "This player is blacklisted."
  end

  -- Submit the rank request
  local res = HttpService:RequestAsync({
    Url    = "https://api.roranks.com/app",
    Method = "POST",
    Headers = { ["Content-Type"] = "application/json" },
    Body   = HttpService:JSONEncode({ key = API_KEY, user = tostring(userId), rank = rank })
  })
  return res.StatusCode == 202, "Ranking has been queued."
end
05 Architecture

Queue System

RoRanks runs a global first-in first-out queue that every user shares. All ranking operations happen one at a time, which keeps Roblox's API from getting flooded and makes sure your requests do not fail randomly.

When you POST to /app or /pass the request goes into the queue and you immediately get a 202 back. If Roblox sends back a rate limit error the request gets placed back at the front of the queue and retried after a delay. That delay starts at 5 seconds and doubles each time, up to a maximum of 5 minutes. Once the rank goes through you get a notification on your Discord webhook either way.

⚠ Do not assume the rank has been applied the moment you receive a 202. Watch your webhook for confirmation or check the player's group rank through the Roblox API if you need it right away.
06 Reference

Error Messages

Because ranking happens asynchronously these errors get sent to your Discord webhook rather than coming back in the HTTP response. Each error embed shows the affected user ID, group ID, and how many rankings have been processed so far.

Plan Limit Reached You have used your entire monthly quota for that type. It resets on the same date each month based on when your key was created.
Invalid Roblox Account The Roblox cookie on your key is no longer valid. Reach out to the RoRanks team to get your credentials updated.
Insufficient group permissions The ranking account does not have permission to change ranks in your group. Make sure it has the Manage Group Members permission turned on.
This user is not in the group The player has not joined your Roblox group yet. They need to be a member before they can be ranked.
This user already has the rank The player is already at the rank you tried to give them so nothing changed. No quota is used up for this.
Invalid rank for this group The rank value you sent does not match anything in your group. Double check the rank number, rolesetId, or role name you are passing.
Failed to set rank Something unexpected went wrong on the Roblox side. This is usually temporary and the request will have already been retried before this message appears.

HTTP status codes

  • 400 Key is expired or a required field is missing from the request body
  • 401 The password or credentials were wrong or missing
  • 404 The key was not found or has already expired
07 Webhooks

Discord Webhooks

After every ranking attempt RoRanks sends a rich embed to the Discord webhook saved on your key. The embed includes a generated rank card image showing the player's avatar, the RoRanks logo, and your group icon.

Success Embed Shows up green with the user ID, group ID, and total ranked count. The rank card shows connected gradient lines between all three icons.
Error Embed Shows up red with the failure reason alongside the user and group info. The rank card has a red cross over the group icon.
ℹ Your webhook URL is set when your key is created. To change it reach out to the RoRanks team. Webhook delivery failures are caught silently and will not stop the ranking from going through.
08 Plans

Plans and Quota

Your key has two separate monthly quotas. One tracks Quiz Center rankings and the other tracks GamePass Center rankings. Both reset on the same calendar date every month based on when your key was first created.

TypeCounterResetsEnterprise
appcurrentapp out of maxappMonthlyUnlimited
passcurrentpass out of maxpassMonthlyUnlimited

Enterprise keys have no practical limit. The /check-key endpoint returns "enterprise": true for those keys.

⚠ Once your key hits its expiry date it deactivates automatically and a notification goes to your webhook. Head to discord.gg/tmGheDXbeP to renew or upgrade.
09 Examples

Code Examples

JavaScript using fetch

async function rankUser(userId, rank) {
  const res = await fetch('https://api.roranks.com/app', {
    method:  'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ key: 'your-key', user: String(userId), rank: String(rank) })
  });
  return res.json();
}

Python

import requests

def rank_user(user_id, rank):
    r = requests.post("https://api.roranks.com/app", json={
        "key": "your-key",
        "user": str(user_id),
        "rank": str(rank)
    })
    print(r.status_code, r.json())

Roblox Luau GamePass gate

local MPS         = game:GetService("MarketplaceService")
local HttpService = game:GetService("HttpService")
local Players     = game:GetService("Players")

local PASS_ID = 123456789
local API_KEY = "your-api-key"
local RANK    = "5"

Players.PlayerAdded:Connect(function(player)
  if not MPS:UserOwnsGamePassAsync(player.UserId, PASS_ID) then return end

  HttpService:RequestAsync({
    Url    = "https://api.roranks.com/pass",
    Method = "POST",
    Headers = { ["Content-Type"] = "application/json" },
    Body   = HttpService:JSONEncode({
      key = API_KEY, user = tostring(player.UserId), rank = RANK
    })
  })
end)
